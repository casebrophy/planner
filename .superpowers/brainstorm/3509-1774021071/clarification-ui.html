Card-Focused Session

One card at a time, full width. Progress dots at bottom. Resolve/snooze/dismiss actions inline. Designed for quick decisions — under 5 seconds per card.

Components to Build

ClarificationCard

Renders a single item by kind. Shows question, context, source info, and action buttons. Each kind has its own action layout.

ClarificationSession

Manages the queue: fetches pending items, tracks position, handles resolve/snooze/dismiss API calls, advances to next card.

ClarificationView

Route view wrapping ClarificationSession. Shows empty state when queue is clear. Badge count in sidebar nav.

Supporting work

clarificationService.ts, useClarificationStore (Pinia), sidebar badge count, route registration, TypeScript types for clarification models.
